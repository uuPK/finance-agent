# Finance Agent Windows 新电脑部署指南

本文用于在一台全新的 Windows 10/11 64 位电脑上，从零运行 Finance Agent。完成后可以在浏览器打开实时智能问数工作台，并使用本地 PostgreSQL、合成演示数据和 DeepSeek 模型服务。

本文以项目安装在 `D:\finance-agent` 为例，避免占满系统盘。命令均在 **PowerShell** 中执行。

## 1. 运行架构和端口

| 服务 | 技术 | 地址/端口 | 用途 |
| --- | --- | --- | --- |
| 前端 | React + Vite | `http://127.0.0.1:5173` | 业务用户工作台 |
| 后端 | FastAPI + Uvicorn | `http://127.0.0.1:8000` | Agent、SSE、导出、评测 API |
| 数据库 | PostgreSQL 16 Docker 容器 | `127.0.0.1:5432` | 业务数据、元数据、审计和评测记录 |
| 大模型 | DeepSeek API | `https://api.deepseek.com` | QueryPlan、SQL、语义审核和答案生成 |

前端会将 `/api` 请求自动代理到 `http://127.0.0.1:8000`，因此本地开发不需要单独配置前端 API 地址。

## 2. 前置条件

请使用 Windows 管理员账户或具备安装软件权限的账户。首次安装会下载 Docker 镜像、Python 包、Node 包和模型调用依赖，需要可访问互联网。

需要安装：

- Git 2.4x 或更高版本
- Docker Desktop（启用 WSL 2 后端）
- Python 3.11 或更高版本，以及 `uv`
- Node.js LTS（建议 20 或 22）和 npm
- 一个可用的 DeepSeek API Key

### 2.1 安装基础软件

用管理员身份打开 PowerShell，依次运行：

```powershell
winget install --id Git.Git --exact --source winget
winget install --id Docker.DockerDesktop --exact --source winget
winget install --id OpenJS.NodeJS.LTS --exact --source winget
winget install --id astral-sh.uv --exact --source winget
```

如果 Docker Desktop 提示需要 WSL 2，请在管理员 PowerShell 中执行以下命令并按提示重启：

```powershell
wsl --install
```

安装结束后关闭并重新打开 PowerShell，确认命令可用：

```powershell
git --version
docker --version
uv --version
node --version
npm --version
```

### 2.2 将 Docker 数据放到 D 盘

启动 Docker Desktop，完成首次初始化后：

1. 打开 `Settings` -> `Resources` -> `Advanced`。
2. 将 `Disk image location` 改为 `D:\DockerDesktop`。
3. 点击 `Apply & restart`。

这样 PostgreSQL 镜像和 Docker 卷会保存在 D 盘。继续前请确保 Docker Desktop 的状态为 `Engine running`。

## 3. 获取项目和配置密钥

### 3.1 克隆项目到 D 盘

```powershell
Set-Location D:\
git clone https://github.com/uuPK/finance-agent.git finance-agent
Set-Location D:\finance-agent
```

若仓库由其他方式提供，请将完整项目目录放到 `D:\finance-agent`，并确保其中存在 `backend`、`frontend`、`docker-compose.yml` 和 `.env.example`。

### 3.2 创建 `.env`

不要复制其他电脑的 `.env` 文件，也不要将 API Key 提交到 Git。复制模板并编辑：

```powershell
Copy-Item .env.example .env
notepad .env
```

至少设置以下内容。将 `your_deepseek_api_key` 替换为自己的密钥；密钥不要加引号，也不要留下空格。

```env
APP_NAME=Finance Agent
ENVIRONMENT=local

DATABASE_URL=postgresql+psycopg://finance_agent:finance_agent@localhost:5432/finance_agent

LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-chat
LLM_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=your_deepseek_api_key
LLM_TIMEOUT_SECONDS=30
MAX_RETRY=2

SQL_TIMEOUT_SECONDS=30
MAX_RESULT_ROWS=1000
RESULT_PREVIEW_ROWS=100
EXPORT_MAX_ROWS=100000
EXPORT_TIMEOUT_SECONDS=60

BACKEND_CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

公司网络需要 HTTP 代理时，再填写 `LLM_PROXY_URL`，例如 `http://127.0.0.1:7890`。不使用代理时保持为空。

## 4. 初始化本地数据库和演示数据

以下步骤会创建本机 PostgreSQL 容器、表结构、实时事件表、人工复核表、导出审计表，以及 500 个客户的合成演示数据。

> 警告：`--reset` 会清空本机容器中 `mart` 与 `metadata` 下的合成数据后重新生成。不要对未来接入的正式比赛数据或生产数据执行该命令。

在项目根目录 `D:\finance-agent` 执行：

```powershell
docker compose up -d postgres
docker compose ps
```

输出中 `finance-agent-postgres` 应为 `running` 或 `healthy`。然后初始化数据库：

```powershell
Get-Content -Raw backend\db\schema.sql |
  docker exec -i finance-agent-postgres psql -U finance_agent -d finance_agent -v ON_ERROR_STOP=1

docker cp backend\db\migrations\002_query_events.sql finance-agent-postgres:/tmp/002_query_events.sql
docker exec finance-agent-postgres psql -U finance_agent -d finance_agent -v ON_ERROR_STOP=1 -f /tmp/002_query_events.sql

docker cp backend\db\migrations\003_evaluation_review.sql finance-agent-postgres:/tmp/003_evaluation_review.sql
docker exec finance-agent-postgres psql -U finance_agent -d finance_agent -v ON_ERROR_STOP=1 -f /tmp/003_evaluation_review.sql

docker cp backend\db\migrations\004_query_exports.sql finance-agent-postgres:/tmp/004_query_exports.sql
docker exec finance-agent-postgres psql -U finance_agent -d finance_agent -v ON_ERROR_STOP=1 -f /tmp/004_query_exports.sql

docker cp backend\db\migrations\005_review_scope_and_query_feedback.sql finance-agent-postgres:/tmp/005_review_scope_and_query_feedback.sql
docker exec finance-agent-postgres psql -U finance_agent -d finance_agent -v ON_ERROR_STOP=1 -f /tmp/005_review_scope_and_query_feedback.sql
```

安装后端依赖并写入合成数据：

```powershell
Set-Location D:\finance-agent\backend
uv sync
uv run python db/seed_synthetic_data.py --reset --customers 500 --days 180
```

成功后可用以下只读命令检查客户数：

```powershell
docker exec finance-agent-postgres psql -U finance_agent -d finance_agent -c "select count(*) as customer_count from mart.customer_info;"
```

结果应为 `500`。

## 5. 安装前端依赖

打开第二个 PowerShell 窗口并运行：

```powershell
Set-Location D:\finance-agent\frontend
npm ci
```

`npm ci` 会严格根据仓库中的 `package-lock.json` 安装已锁定版本；首次执行需要几分钟。

## 6. 启动系统

需要保留两个 PowerShell 窗口，不要关闭正在运行的服务。

### 窗口 A：启动后端

```powershell
Set-Location D:\finance-agent\backend
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000
```

出现以下内容表示后端已经启动：

```text
Uvicorn running on http://127.0.0.1:8000
```

在第三个 PowerShell 窗口验证：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/health
Invoke-RestMethod http://127.0.0.1:8000/api/llm/status
```

第一个命令应返回 `status: ok`；第二个命令应显示已启用 LLM 且不会返回密钥。

### 窗口 B：启动前端

```powershell
Set-Location D:\finance-agent\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

打开浏览器访问：<http://127.0.0.1:5173>

首页就是智能问数工作台。可先输入：

```text
当前资产大于100万的客户有多少
```

页面应立即创建查询记录，并通过 SSE 显示元数据检索、QueryPlan、SQL 审核、SQL 执行和结果审核阶段。最终结果默认预览最多 100 行；完整结果可从查询记录导出 Excel、CSV 或 JSON。

## 7. 可选质量检查

```powershell
Set-Location D:\finance-agent\backend
uv run --with pytest --with pytest-asyncio python -m pytest

```

## 8. 停止和再次启动

停止前后端服务：在各自 PowerShell 窗口按 `Ctrl+C`。

停止数据库但保留数据：

```powershell
Set-Location D:\finance-agent
docker compose stop postgres
```

下次启动只需：

```powershell
Set-Location D:\finance-agent
docker compose up -d postgres

# 新开窗口启动后端
Set-Location D:\finance-agent\backend
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000

# 再新开窗口启动前端
Set-Location D:\finance-agent\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

仅在确定要删除本机 Docker 数据库时，才执行以下破坏性命令：

```powershell
docker compose down -v
```

执行后必须重新完成第 4 节的数据库初始化。

## 9. 常见问题

### Docker Desktop 未启动或 PostgreSQL 无法连接

确认 Docker Desktop 显示 `Engine running`，然后执行：

```powershell
Set-Location D:\finance-agent
docker compose ps
docker compose logs postgres --tail 100
```

若 `5432` 已被其他 PostgreSQL 占用，可先停止占用进程，或修改 `docker-compose.yml` 的宿主机端口并同步修改 `.env` 中 `DATABASE_URL`。

### `uv`、`git`、`docker` 或 `npm` 命令找不到

关闭所有 PowerShell 窗口后重新打开；若仍失败，检查软件是否完成安装并确认其安装目录已加入系统 `PATH`。

### 前端能打开但请求报错或一直转圈

确认后端窗口仍在运行，并访问 `http://127.0.0.1:8000/api/health`。前端必须运行在 `5173`，后端必须运行在 `8000`，以匹配 Vite 代理和 CORS 配置。

### 大模型调用失败

依次检查：

1. `.env` 中 `DEEPSEEK_API_KEY` 已填入且没有引号、空格或换行。
2. `Invoke-RestMethod http://127.0.0.1:8000/api/llm/status` 显示已启用 LLM。
3. 当前网络可访问 `https://api.deepseek.com`；受限网络需要正确填写 `LLM_PROXY_URL`。
4. API Key 的余额、权限和模型服务状态正常。

修改 `.env` 后必须重启后端服务。

### 端口被占用

查看端口占用：

```powershell
Get-NetTCPConnection -LocalPort 5432,8000,5173 -ErrorAction SilentlyContinue |
  Select-Object LocalPort, State, OwningProcess
```

结束不再需要的进程，或在同时修改前端代理、`.env` CORS 和 Docker 映射后使用新的端口。建议开发机保持默认端口，减少配置差异。

# Demo Script

## 演示目标

展示系统如何从自然语言问题出发，经过 QueryPlan、Guardrail 和 Actor-Critic 修复闭环，最终得到可信查询结果。

## 推荐场景

1. 简单查询

   查询当前资产大于 50 万的客户数量。

2. 中等查询

   查询近三个月交易次数超过 3 次且当前资产大于 50 万的客户列表。

3. 复杂查询

   找出近三个月资产净流入明显、尚未持有基金产品、由同一服务经理维护的客户，并按服务经理汇总人数。

## 展示顺序

1. 输入自然语言问题。
2. 展示识别出的业务意图。
3. 展示元数据召回结果。
4. 展示 QueryPlan。
5. 展示 Critic 检查与修复建议。
6. 展示最终 SQL 和 Guardrail 检查结果。
7. 展示查询结果和自然语言解释。
8. 展示评测中心中的准确率和失败原因分布。
